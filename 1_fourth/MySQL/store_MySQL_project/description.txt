Проект базы данных представляет собой интернет-магазин. В базе представленны 12 таблиц.

В проекте я постарался учесть (в скобках будет указано название таблицы):
- пользователей (users), их профили (profiles);
- заказы пользователей (orders) и их содержимое (order_products)
- каталог товаров (products), их категории (product_category) и описание товаров (products_desc);
- отзывы (reviews) и лайки для товаров (likes);
- скидки на товары (discounts)$
- медиа файлы для товаров (media) и их категории (media_types).

Проект решает задачи по созданию заказов, привязка заказов к пользователям и купленным товарам. Так же
имеется каталог для просмотра товаров и категории товаров для их фильтрации. Так же отзывы, для того, чтобы
пользователь мог понять насколько конкретный товр хорош.

Было еще много идей для таблиц (служебные, для статистики и т.д.), но в пределах данного урока
решил их не использовать, так как время ограниченно и есть еще много чего хотелось бы учесть.

В запросах где необходимо вводить id пользователя лучше вводить id = 28, так как там сгенерированные данные
немного подправлены для наглядности демонстрации результата запроса.

Так же из-за невозможности сгенерировать корректно данные присутствует небольшая "коллизия", например
статус заказа "executed" (исполнен), а статус доставки может быть "packaging" (упаковывается).
